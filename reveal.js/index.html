	$("livingroom").bind("motion", function() {

Esse trecho de codigo mostra bem o que é pretendido com a filosofia nodebots (Perguntar se alguém consegue entender o que o código faz)

A idéia por trás do Nodebots é criar abstrações mais próximas ao mundo real (Explicar código)

Web Assembly (WASM)

WASM

Definição de "assembly language": é uma linguagem de programação de baixo nível para um computador, ou outros dispositivos programaveis, que possui uma correspondência muito forte com as instruções da arquitetura da máquina.

JavaScript é uma linguagem de programação de alto nível. Ela é interpretada para bytecode que, por sua vez, é traduzido para linguagem de máquina. Cada browser - navegador - possui a sua VM que "entende" JavaScript.

JavaScript - VM ubíqua instalada em todas as máquinas

Um ótimo "alvo" de compilação

Algumas linguagens...

Elas são transformadas e não compiladas e, além disso, tendem a gerar códigos JavaScript diversos.

ASM.JS

sub-conjunto do JavaScript

Pensando nisso, por que não ter como "alvo" um subconjunto do JavaScript OTIMIZADO? Com foco em performance?

It’s a specialized subset of JavaScript that’s optimized for performance, especially for this use case of applications compiled to JavaScript

Características e Limitações

ASM.JS só é capaz de manipular uma seleção de diferentes tipos de números e nenhum outro tipo de dado ou estrutura de dados (incluindo strings, booleans, ou objects).

Typed Arrays

Ex.: substitui toda variável global por um objeto HEAP, que é um objeto Typed Array, que é muito otimizado para performance.

Isso faz com que o código asm seja traduzido diretamente para assembly, sem ter que interpretá-lo.

Como a compilação funciona?

C/C++ => LLVM => Emscripten => JavaScript

Os otimizadores LLVM usam informações de tipo para realizar otimizações úteis. Tais otimizações estão disponíveis somente para código compilado. Fazer isso diretamente em JavaScript não seria prático e seria difícil de manter.
